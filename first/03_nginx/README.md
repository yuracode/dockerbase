# 03 — Nginx をコンテナで起動する

Web サーバ Nginx を 1 コマンドで起動し、ブラウザから確認します。

---

## 実行コマンド

```bash
docker container run -p 8080:80 -d --name my-nginx nginx:alpine
```

実行すると、英数字の長い ID（コンテナ ID）が表示されて、プロンプトに戻ります。
バックグラウンドで Nginx が動き出している状態です。

---

## オプションの意味

| オプション | 意味 |
|-----------|------|
| `-p 8080:80` | ホストの 8080 番 → コンテナの 80 番 にポート転送 |
| `-d` | detach — バックグラウンドで起動 |
| `--name my-nginx` | コンテナに名前をつける（後で操作しやすい） |
| `nginx:alpine` | Alpine 版の軽量 Nginx イメージ |

---

## 動作確認

ブラウザで以下を開きます：

**[http://localhost:8080](http://localhost:8080)**

「Welcome to nginx!」のデフォルトページが表示されれば成功です。

---

## ポートマッピングの図解

```
[ブラウザ]
   │  http://localhost:8080
   ▼
┌──────────────────────┐
│ ホストPC             │
│   8080番 ─────┐      │
│               │      │
│   ┌───────────▼───┐  │
│   │ コンテナ       │  │
│   │   80番 (Nginx) │  │
│   └───────────────┘  │
└──────────────────────┘
```

- **左（ホスト）側は好きな番号でOK**（他と被らなければ）
- 右（コンテナ）側は Nginx が待ち受ける 80 番で固定
- 例：`-p 8888:80` ならブラウザは `http://localhost:8888`

---

## 起動中のコンテナを確認

```bash
docker container ls
```

`my-nginx` が `STATUS: Up` で表示されていればOK。

---

## コンテナを停止・削除する

```bash
# 停止
docker container stop my-nginx

# 削除
docker container rm my-nginx
```

停止してから削除、が基本の順序です。
